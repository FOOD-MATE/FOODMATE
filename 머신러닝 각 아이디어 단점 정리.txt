코사인 유사도를 이용한 추천[키워드] 단점
1. 상위 추천 데이터의 경향 쏠림

필요한 추가 기능
-> 추천 데이터를 평준화하는 기능
: 생각해본게 분위를 임의로 나누어서 랜덤하게

로지스틱 회귀만을 이용한 추천 단점
1. 학습은 정상적으로 이루어지나 테스트 단계에서 최소 샘플 수를 구하는게 어려움(3만개이상)
2. 영양소 기반이기에 데이터가 다양하게 나오나 해당 데이터가 유저가 원한다고 예측하기에는 어려움

위에다가 데이터가 유저가 원한다고 생각할 수 있는 제약 조건
1. 코사인 유사도에서 음식 이름을 2만개 추출해가지고 영양소 기반 추천값이랑 코사인 유사도에서 추출한 음식의 이름이랑 같을때?

조건을 받는다고 할떄
1. 카테고리
2. 키워드 중 선택

조건을 안받으면 영양소 기반으로 랜덤 추천(=> 가장 비슷한 유형의 음식이 나옴)
1. 기반 데이터 없이 사용자가 원하는 fat, carbo, protein 값을 어떻게 예측해서 샘플 데이터 갯수만큼 나올건지?


필요한 추가 기능
-> 샘플을 어떠한 기준에 맞추어서 만들어야함
-> 유저가 원하는 경향의 데이터에 맞추는 어떠한 기준을 만들어야함

양쪽을 섞어쓸때
-> 그냥 어려움, 예측 결과 예상할 수 없음


유저기반? 예측값기반 랜덤추천?

유저가 선택한 우리의 예측값
탄수화물 14%, 지방 34%, 단백질 52%
==> 이러한 비율과 가장 유사한 음식들이 추천

이거를 만드는 식을 구해서 랜덤난수를 곱하든 제곱을 하든 해서 나오는 샘플 데이터를 로지스틱회귀 모듈에 입력
